# CLAUDE.md

このファイルは Claude Code がプロジェクトを理解するためのコンテキストファイルです。
パート1（動作原則）はデフォルトで有効です。パート2（プロジェクト情報）をプロジェクトに合わせてカスタマイズしてください。

---

## パート1: 動作原則

### ワークフロー

- 3ステップ以上または設計判断が必要なタスクはプランモードで開始する
- 問題が発生した場合は推し進めず、即座に再計画する
- 変更の動作実証（テスト実行・ビルド確認など）なしに完了を宣言しない

### サブエージェント活用

- 調査・探索・並列分析はサブエージェントに委譲してメインコンテキストを保持する
- 1タスク＝1サブエージェント体制で効率的に並列処理する

### 品質基準

- 最小限のコード影響で変更を実装する（単純性優先）
- 根本原因を特定する（安易な回避策を取らない）
- 「シニアエンジニアがこのコードを承認するか？」を自問する
- 非自明な変更時に「より優雅な方法はないか」を検討する（ただし過度設計は避ける）

### タスク管理

- `tasks/todo.md` に確認可能な項目として計画を記述する
- 進捗を逐次マークする
- 各ステップで変更内容を説明する

### 自己改善

- ユーザーからの指摘後は `tasks/lessons.md` を更新する
- 再発防止ルールを作成し、セッション開始時に確認する

### 自律的バグ修正

- バグ報告を受けたら直ちに修正に取りかかる
- ログ・エラー・失敗テストを特定して解決する
- CI の失敗テストも自発的に修正する
